# napsnap 概要（評価用サマリー）

> 顔なし・24時間で消える・"生活の痕跡" を共有する小さなSNS。
> Webデモ：https://tmk4men.github.io/napsnap/

---

## 1. ひとことで

**「顔を出さない、24時間で消える、日常SNS」**
盛らない・盛らせない設計で、写真を上げる心理コストを極限まで下げた“素のままで”の場所。号外（新聞紙面）のデザイン語彙×白黒の配色で、見た目もブランドも他のSNSと意図的にズラしている。

## 2. ポジショニング

大手SNS（X / Instagram / Threads / BeReal）と同じ土俵では戦わない前提。

| | 大手 | **napsnap** |
|---|---|---|
| 価値 | 広く・盛る・残る | **狭く・素のまま・消える（けど"あとで思い出になる"）** |
| 顔出し | 推奨 | **不可**（投稿前に顔検知でブロック） |
| 期限 | 永続 | **24時間で消える** |
| 配信 | アルゴ・おすすめ | **フォロー＋お題（共通テーマ）の2層** |

「盛り疲れ／顔出し疲れ」が競合の隙間。**消える × 思い出の堀**（過去の自分の投稿は端末ローカルに残る）で、軽く出せて後で宝物になるループを回す。

## 3. コア仕様

### 投稿
- 1投稿 = 写真 + 1.8秒の環境音 + 任意の短い手書き風キャプション
- 写真は WebP / JPEG q0.72 / 長辺1080px に自動圧縮（≈150-200KB）
- 音声はモノ・22kHz・32kbps（≈7KB）
- 期限：通常24時間 / お題はその日の23:59
- 撮影前に顔検知（Web=MediaPipe、ネイティブは将来 react-native-vision-camera 予定）

### 相互アンロック（パス制）
- 他人の投稿は「自分が1枚出すまで」モザイク＋音なし＋投稿者名＋"N人が反応"だけ表示
- 1枚出すと **6時間** くっきり見える（"出す → 見る" のリズム）

### 「お題」タブ（今日の見出し）
- 毎日0時に**全員共通のテーマ**が切り替わる（"ごはん" "窓の外" "コンビニ戦利品" 等16種ローテ）
- 上下スワイプで投稿を1枚ずつ何度でも見返せる
- 左右スワイプで **「フォロー / おすすめ」** を切替
  - フォロー：自分＋フォロー中の人の今日の投稿
  - おすすめ：全実在ユーザーの今日の投稿（**完全公開**）
- お題への投稿はホーム/残したには出ない別世界扱い
- セキュリティ設定で「お題は自分とフォローのみに見せる」も選べる
- 毎朝8時（JST）に "今日の見出し：◯◯" のpush通知が全員に届く（BeReal式の全員同時起動）

### リアクション・足あと
- 反応は3種：💖 すき / 😂 わら / 👀 やば（既読数や❤️数の競争を持ち込まない）
- 自分の投稿への足あと・反応は通知に出る（ベル）
- 「**残した**」タブで、自分が反応した投稿だけ24時間見返せる

### 思い出（端末ローカル）
- 期限切れの自分の投稿は**サーバから消える / 端末ローカルだけに残る**
- 「1年前の今日」をホームに自動表示（朝の再訪フック）
- 月/年のふり返りリール（未実装）

### 公式アカウント `@napsnap`
- 全ユーザーが最初から自動フォロー＝コールドスタート時に「見るものがある」
- アイコンは黒地に白の **「N」**（ブランドフォント DM Serif Display）
- 認証マーク（チェック）は汎用の `is_official=true` で別ユーザーにも付けられる（例：開発者 `@ktomo_dev`）

## 4. デザイン言語

**確定方針：「号外（新聞紙面）」の構成・書体 × 白黒（Twitter/IG的）の配色**

- **題字**：DM Serif Display（高コントラストのディドネ／新聞のマストヘッド）
- **見出し**：Shippori Mincho（詰めた明朝・余韻のコピー）
- **本文／ラベル**：BIZ UDPGothic（公的書類の角ゴ・"通知/告知"の質感）
- **ID・時刻**：DM Mono（等幅で"活字で組んだ"質感）
- **ライト**：白地に黒 / **ダーク**：黒地に白
- 強調はすべて黒（ダークは白）。**赤は warn のみ**（通知バッジ・残り時間）
- 写真は「チェキ」スタイル（細罫で囲った白縁＋下に手書きキャプション）
- 角丸ほぼ無し（紙面は直角）。影は基本使わず、罫線で階層を表す

## 5. 技術スタック

- **フロント**：Expo SDK 56 / React Native 0.85 / React 19 / TypeScript（strict）
- **状態管理**：Zustand + persist（AsyncStorage）
- **配信**：GitHub Pages（Webデモ自動デプロイ）／ EAS（実機ビルド予定）
- **バックエンド**：Supabase
  - DB（Postgres + RLS）：profiles / follows / posts / reactions / views / push_tokens
  - Auth：匿名サインイン（永続）
  - Edge Functions：`send-push` / `prune-expired` / `topic-broadcast` / `r2-upload`
- **メディア配信**：**Cloudflare R2**（egress無料の Object Storage / S3互換）。Supabase Storageはフォールバック。
- **push**：Expo Push（FCM/APNs を内包）→ pg_cron + Edge Function 経由で発火
- **顔検知**：Web=MediaPipe Tasks Vision（撮影前ブロック）
- **広告**：Google AdMob 採用予定（Rewarded／Native／Banner の3面、ネイティブ実機ビルド後に配線）

### 同期方式（コストの肝）
- **段階1 ポーリング廃止済**：従来の15秒タイマーを撤廃し、`focus/tab切替/前面復帰/操作後` の駆動に変更。前面にいる間だけ90秒の緩い取り込みでフォールバック。
- **段階2 push（実機ビルド後に有効化）**：投稿・反応で DB トリガが Edge Function を叩いて Expo Push 発火。クライアント側コード・SQLトリガ・Webhook・関数 すべて配線済み。実機 dev ビルドで token 登録すれば即稼働。

### コスト最適化（実装済み）
- R2移行（配信費の主軸が実質ゼロ化）
- 反応数・足あと数の denormalization（`posts.reaction_count` / `view_count` 列＋トリガ）でメタ取得を最小化
- 期限切れ投稿の自動掃除（15分おきに R2/Supabase Storage/posts 行をまとめて削除）
- プロフィール取得を id 指定に絞る（全件 select の廃止）
- 思い出は端末ローカル＝サーバ側は24h分しか溜まらない

## 9. 評価してほしい観点（参考）

- 「**消える × 思い出**」の二項対立を物語にできているか
- 顔出し疲れ／盛り疲れの代替として、コア体験は十分か
- お題タブの完全公開（フォロー/おすすめ）の発見動線とプライバシーのバランス
- 弱気ARPU=¥22.5 で本当にインフラ費を上回るか
- "号外×白黒"のブランド統一感が、SNSとして見やすさを損ねていないか
- コールドスタート（友達0人状態）でも続けたいと思える設計になっているか
- 通報・モデレーションの薄さは、24時間消滅の防御で本当に十分か
