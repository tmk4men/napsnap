# napsnap 企画書

> このドキュメントは napsnap の正式な企画書（仕様の出発点）である。
> 実装はこの企画書の「コア体験」と「やらないこと」を最優先で守る。

## 1. アプリ概要

**napsnap** は、顔や人を映さずに、友達の“生活の痕跡”を共有する少人数向けSNS。

ユーザーは自分の生活を1枚投稿すると、**6時間だけ友達の投稿を閲覧できる**。
閲覧は通常のタイムラインではなく、1投稿ずつ全画面で表示される一方通行フィード。
リアクションせずにスワイプした投稿は自分の画面から消え、リアクションした投稿だけが24時間見返せる。

コンセプトは、

> **1枚出すと6時間ひらく、顔なし日常SNS。**

または、

> **人が映ると撮れないカメラで、友達の日常をのぞく。**

---

## 2. プロダクト思想

napsnapは、Instagramのような映えSNSではない。
BeRealのように自分の顔や今の姿を撮るSNSでもない。
TikTokのように知らない人の投稿を無限に消費するSNSでもない。

napsnapは、

> **顔ではなく、生活でつながるSNS**

である。

ユーザーが撮るのは、人ではなく以下のような生活の一部。

* 机
* 足元
* 飯
* 飲み物
* 部屋の一部
* 玄関
* カバン
* 寝床
* 床
* 手元
* サボっている痕跡

人が映らないことで、逆にその人の生活感や癖が見える。

---

## 3. コア体験

### 基本ループ

1. アプリを開く
2. 友達の投稿はロックされている
3. 自分の生活を1枚撮影して投稿する
4. 6時間だけ友達の投稿フィードが開く
5. 友達の投稿を1枚ずつ見る
6. リアクションするか、スワイプして流す
7. リアクションした投稿だけ24時間見返せる
8. 自分の投稿には足跡とリアクションが残る
9. 6時間後に再びロックされる
10. また見たければ、もう1枚投稿する

---

## 4. MVPで作る機能

### 4.1 認証・ユーザー

**必須**

* ユーザー登録 / ログイン / 表示名設定 / 最低限のプロフィール

**プロフィール項目**

* 表示名 / アイコン（初期アイコン可）/ ユーザーID

**作り込まないもの**

* 長文プロフィール / 自己紹介 / フォロワー数表示 / 公開プロフィールページ / プロフィール写真の強い推奨

顔出ししない思想のため、プロフィール写真は必須にしない。

### 4.2 グループ機能

napsnapは公開SNSではなく、少人数グループSNSとして始める。

**必須**: グループ作成 / 招待リンクで参加 / メンバー一覧 / 退出 / 最大人数制限
**推奨人数**: 3〜8人
**MVP制限**: 最初は1ユーザー1グループでもよい。公開検索は不要。

### 4.3 カメラ投稿

**必須**: アプリ内カメラ / 撮影 / 撮影後プレビュー / 投稿ボタン / 再撮影 / 投稿後に6時間閲覧パスを付与

**投稿ルール**: 投稿は24時間残る。24時間後に自動削除（非表示）。1回1枚。動画はMVPでは不要。

### 4.4 人間禁止カメラ（最大の差別化要素）

**目的**: 人を映さず、生活の痕跡を撮る文化を作る。

**理想ルール**: 顔検知＝シャッター不可 / 明確な人物＝警告 / 手足・影＝許容 / ポスターやテレビ内の人物＝許容寄り。

**MVP簡易版**: 顔検知のみ。顔が検出されたら撮影ボタンを無効化し「顔は禁止」と表示。

**カメラ画面の文言**: 「人間なしで、今を1枚」「顔は禁止」「生活だけ撮れ」「人が映ってるかも」「痕跡を撮れ」

**注意**: これは安全保証ではなく、文化形成のための制約。「人が映らないから安全」とは表現しない。

### 4.5 6時間閲覧パス（中心ルール）

ユーザーが1枚投稿すると、現在時刻から6時間、友達の投稿を閲覧できる。

* **Locked**: 友達の投稿は見られない。ロック画面を表示。投稿すると6時間開くことを伝える。
* **Open**: 友達の投稿フィードを閲覧できる。残り時間を表示してもよい。6時間経過後 Locked に戻る。

ロック画面コピー候補:
> 閉じてる / 1枚撮ると、6時間だけ友達の痕跡が見える。
> 友達の今はロック中 / 君の1枚で開く。
> 3人の痕跡が待ってる

### 4.6 閲覧フィード

通常のタイムラインではなく、1投稿ずつ全画面で表示する。

* 1枚ずつ全画面 / 一方通行スワイプ / 一度流した投稿には戻れない
* リアクションした投稿だけ後から見返せる
* 未リアクションのままスワイプした投稿は、閲覧キューから消える

重要: TikTokのような無限フィードではない。友達の投稿だけを短時間で濃く見る。

### 4.7 リアクション

「いいね」ではなくリアクション。映え写真ではないので、軽い反応の方が合う。

候補: 👀 見た / 😂 笑う / 🫠 わかる / 🌀 やば / 🫶 すき / 🟡 napsnap独自

* 1投稿につき1リアクション
* リアクションした投稿は「残した」タブに入り24時間見返せる
* リアクション内容は投稿者に表示される

### 4.8 残したタブ

リアクションした投稿だけを見返せる場所。

* リアクション済みのみ / 24時間で消える / このタブ内は上下自由（一方通行制限なし）
* タブ名: MVPでは **「残した」**

### 4.9 自分タブ

自分の投稿と、それに対する反応を確認する場所。

表示: 自分の投稿 / 残り表示時間 / 足跡 / リアクション一覧
例: 「12人が見た」「5人が反応した」「😂 みほ」「👀 たくや」「🫠 けん」

表示しない: 誰がスルーしたか / スルー人数 / 未リアクション者への圧
（スルーを可視化すると人間関係が重くなるため）

### 4.10 足跡

投稿者には誰が見たかが分かる。

* 閲覧者一覧 / リアクションした人はリアクションつき / 見ただけの人は足跡 / 「スルー」という言葉は使わない
* 足跡だけでは通知しない。リアクションが来た時だけ通知してよい。

### 4.11 通知

BeRealのようなランダム強制通知は採用しない。
napsnapは通知で撮らせるのではなく、**見たいから自分も1枚出す**アプリ。

**ロック後通知**（条件: パス切れ＋友達投稿あり＋未再投稿）
> みんな、君の今を待ってる / 1枚撮ると6時間ひらく

**リアクション通知**（条件: 自分の投稿に反応）
> みほが君の痕跡に反応した / 君の1枚が残された

**斬る通知**: ランダム強制 / お題 / 全員同時 / 2分以内 / 何度も催促 / 罰っぽい通知

## 5. 画面構成

* **オンボーディング**（最大3画面）: ①人が映ると撮れない、日常SNS。 ②1枚撮ると、6時間だけ友達の痕跡が見える。 ③反応した投稿だけ、24時間残る。 → グループ作成/招待参加へ
* **ホーム/ロック**: ロック状態 / 投稿ボタン / 友達の投稿人数 / コピー「閉じてる」 / ボタン「今を撮る」
* **カメラ**: カメラビュー / 撮影ボタン / 人検知状態 / ガイド「人間なしで、今を1枚」/ 顔検知時「顔は禁止」
* **投稿プレビュー**: 写真 / 投稿 / 再撮影 → 投稿後6時間パス付与しフィードへ
* **フィード**: 1枚ずつ全画面 / 投稿者名 / 時刻 / リアクション / スワイプで次へ（戻れない、未反応で進むとキューから消える）
* **残したタブ**: リアクション済み / 24時間以内 / 自由に上下移動
* **自分タブ**: 自分の投稿 / 閲覧者 / リアクション / 残り時間

## 6. データ設計イメージ

* **User**: id / displayName / avatar / createdAt
* **Group**: id / name / memberIds / inviteCode / createdAt
* **Post**: id / userId / groupId / imageUrl / createdAt / expiresAt / humanDetectionStatus / viewCount / reactionCount
* **View**: id / postId / viewerUserId / viewedAt
* **Reaction**: id / postId / userId / reactionType / createdAt
* **AccessPass**: id / userId / groupId / openedAt / expiresAt
* **FeedState**: userId / postId / status(unseen/viewed/skipped/reacted) / updatedAt

## 7. MVPでやらないこと

公開タイムライン / フォロー / おすすめユーザー / コメント / DM / 動画 / 位置情報 / お題 / ランダム強制通知 / 全員同時投稿 / 2分制限 / 長文プロフィール / ハッシュタグ / 検索 / AI分析 / いいね数ランキング / スルー表示 / 永久保存 / Web公開ページ

napsnapは機能数で勝つアプリではない。制約と閲覧体験で勝つアプリである。

## 8. UIトーン

外側はシンプル、中身は少しゆるい。余白多め / 丸みのあるUI / 黒・白・ライム系アクセント / 顔出し圧を感じさせない / かわいすぎない / 怖すぎない / “生活の痕跡”が主役。

**推奨カラー**
* 黒: `#050505`
* 白: `#F7F7F2`
* ライム: `#DFFF2F`
* グレー: `#8A8A8A`
* 警告赤: `#FF3B3B`
* クリーム: `#FFF8E8`

**良い文言**: 1枚でひらく / 人間なしで、今を1枚 / 顔は禁止 / 友達の痕跡が待ってる / 反応したものだけ残る / 閉じてる / 君の今を待ってる
**避ける文言**: 素敵な瞬間を共有しよう / 思い出を保存しよう / みんなとつながろう / 投稿しないと見られません / 失敗しました / あなたらしい毎日を

## 9. 重要KPI

* **最重要**: 投稿後6時間以内の閲覧率 / ロック画面からの投稿率 / 投稿頻度 / 7日継続率
* **閲覧体験**: 1回あたり閲覧投稿数 / リアクション率 / スキップ率 / 残したタブ閲覧率
* **SNS性**: 投稿あたり足跡数 / リアクション数 / グループ内アクティブ人数 / 招待成功率
* **通知**: ロック後通知からの投稿率 / 通知オフ率 / リアクション通知からの起動率

---

## 10. 実装優先順位

* **Phase 1（最小SNS）**: 認証 / グループ作成・参加 / 写真投稿 / 24時間表示 / 6時間閲覧パス / ロック画面 / 基本フィード / リアクション / 残したタブ / 自分タブ
* **Phase 2（napsnapらしさ）**: 人検知カメラ / 顔検知時シャッター無効 / 足跡表示 / ロック後通知 / リアクション通知 / 一方通行フィードの完成
* **Phase 3（継続性）**: グループ内の軽いまとめ / 追加リアクション / カメラフレーム / 通知文改善 / 週間の軽い振り返り

---

## 11. 企画上の注意点

**守るべきこと**: 投稿は1枚 / 人を主役にしない / 公開TLを作らない / コメントを作らない / いいね数競争にしない / 反応したものだけ残す / 見たいなら自分も出す / 友達グループ内に閉じる

**最も大事な体験**
> 自分の生活を1枚出す → 6時間だけ友達の生活が開く → 1枚ずつ見て、反応したものだけ残る

この体験が崩れる機能は足さない。

---

## 12. 最終コンセプト文

> napsnapは、顔や人を映さずに生活の痕跡を共有する少人数SNS。
> ユーザーは1枚写真を投稿すると6時間だけ友達の投稿を閲覧できる。
> 投稿は24時間残るが、閲覧者は1投稿ずつ一方通行で見て、リアクションしなければその投稿は自分の画面から消える。
> リアクションした投稿だけは専用タブで24時間見返せる。
> 投稿者には足跡とリアクションが残る。
> BeRealのような強制通知やお題は使わず、「見たいなら自分も1枚出す」という交換感を中心にする。
> MVPでは公開TL、コメント、DM、動画、位置情報、フォロー機能は作らない。
> 目指す体験は「1枚出すと6時間ひらく、顔なし日常SNS」。
